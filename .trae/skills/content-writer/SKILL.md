# 全文撰写 Skill

## 功能概述

本 skill 基于大纲规划和全维度设计方案，自动撰写小说正文内容，确保符合爆款网文的写作风格、节奏要求和爽点设置，生成高质量、高吸引力的章节内容。严格遵循网文读者阅读习惯，最大化留存率和追读率。

## 触发条件

- 用户要求撰写小说正文
- 大纲完成后需要生成章节内容
- 需要批量生成多章节内容
- 工作流接收到大纲后自动触发

## 写作风格指南

### 1. 爆款网文写作核心要素

#### 开篇黄金法则（前三章定生死）
- **第一章立刻出冲突**: 退婚、背叛、羞辱、死亡危机，不要铺垫风景身世
- **第二章立刻给金手指**: 系统、异能、传承、重生记忆，越快越好
- **第三章立刻打脸**: 让看不起主角的人傻眼，读者爽点直接到位
- **每章结尾留悬念**: 敌人反扑、更大危机、神秘声音、关键抉择

#### 语言风格（AI去痕化）
- **通俗易懂**: 避免生僻词汇，用大白话写作
- **口语化**: 对话贴近生活，符合人物身份，避免AI典型痕迹（如英文引号、加粗关键词）
- **简洁明快**: 句子短，节奏快，不拖沓
- **画面感强**: 动作描写具体，场景描写生动
- **去机器化**: 避免AI生成的陈词滥调，如"从此幸福快乐"、"冰山一角"等

#### 人物塑造（防止OOC）
- **目标清晰**: 复仇、变强、守护家人、赚钱、登顶世界
- **性格统一**: 杀伐果断/宠妻狂魔/扮猪吃虎，别摇摆
- **不能憋屈**: 可以暂时弱，但不能长期被踩
- **人物卡片**: 每个人物建立详细档案，确保言行一致

#### 金手指设计（简单有效）
- **简单规则**: 不要复杂规则，最好"自动生效"
- **适度强大**: 能解决问题但不过分无敌
- **有代价**: 使用金手指需要付出相应代价
- **成长性**: 随着剧情推进逐步解锁新功能

### 2. 爆款节奏公式（张弛有度）

#### 爽点密度控制
- **小爽点**: 每3000字一次（约1.5章）
- **中爽点**: 每1万字一次（约5章）
- **大爽点**: 每3-5万字一次（约15-25章）

#### 节奏结构模板
```
【钩子开头】(0-200字)
- 用悬念、冲突或惊人画面吸引读者
- 直接切入场景，不拖泥带水
- 遵循"黄金三秒"原则，开头就有看点

【情节推进】(200-1200字)
- 对话推进为主，描写为辅
- 冲突逐步升级
- 情绪逐步积累
- 每3-5句话要有看点或推进

【高潮爆发】(1200-1700字)
- 爽点爆发/战斗高潮/情感爆发
- 动作描写具体有力
- 情绪释放
- 保证爽感达到峰值

【悬念结尾】(1700-2000字)
- 留下悬念勾读者
- 制造新的期待
- 为下一章做铺垫
- 悬念强度要够，让人忍不住点开下一章
```

#### 节奏变化模式
```
紧张5章 → 放松2章 → 紧张5章 → 放松2章 → 大紧张（高潮）10章 → 放松3章
```

### 3. 描写技巧（提升代入感）

#### 动作描写（增强画面感）
- **具体化**: 不用"他很厉害"，用"他一拳砸出，空气都发出爆鸣"
- **动词精准**: 用"砸"不用"打"，用"劈"不用"砍"，用"撕裂"不用"划破"
- **连续动作**: "他身形一闪，避开攻击，反手一掌拍出，趁对方立足未稳，又是一脚踢出"
- **感官冲击**: "血肉横飞"、"骨裂声"、"空气爆鸣"等

#### 场景描写（营造氛围）
- **五感并用**: 视觉、听觉、嗅觉、触觉、味觉
- **动态描写**: "狂风呼啸，树叶翻飞，沙石漫天"
- **情绪烘托**: 用场景烘托人物心情
- **细节刻画**: "空气中弥漫着淡淡的血腥味"而非"很血腥"

#### 对话描写（个性化和张力）
- **个性化**: 每个人物说话方式不同，符合身份背景
- **有目的**: 每句对话都要推进情节或展现人物
- **有张力**: 对话中暗含冲突或潜台词
- **口语化**: 避免书面语，多用口头禅、感叹词
- **潜台词**: "你来了。"（表面平静，内心波澜）

#### 情感描写（细腻入微）
- **外化情感**: 用动作、表情、语气展现内心，少用直接心理描写
- **细节捕捉**: "他的手微微颤抖"、"她咬住了嘴唇"、"瞳孔急剧收缩"
- **情绪层次**: 从轻微到强烈的情绪递进

### 4. 不同类型章节写法（爆款模板）

#### 战斗章节（爽感最大化）
```
开场: 双方对峙，气氛紧张（200字）
试探: 互相试探实力，展示双方底牌（500字）
交锋: 正式交手，你来我往（800字）
转折: 意外情况/主角爆发/反转（500字）
高潮: 主角爆发，逆转战局（800字）
收尾: 战斗结果，获得奖励，埋下新悬念（200字）
```

#### 升级章节（期待感营造）
```
铺垫: 修炼瓶颈，需要突破（300字）
过程: 修炼过程，痛苦挣扎（600字）
突破: 关键时刻，顿悟突破（400字）
验证: 测试新实力，碾压测试（500字）
收获: 获得新能力/境界提升（200字）
展望: 为下个目标做铺垫（100字）
```

#### 打脸章节（情绪释放）
```
铺垫: 反派挑衅，众人嘲讽（300字）
压抑: 主角隐忍，形势不利（400字）
转机: 机会来临/实力展现（300字）
反转: 主角展露实力，震惊全场（700字）
打脸: 反派被打脸，众人惊呆（300字）
收尾: 主角淡然离去，深藏功与名（100字）
```

#### 感情章节（细腻处理）
```
互动: 男女主自然互动（400字）
暧昧: 制造暧昧氛围（500字）
阻碍: 设置感情阻碍（300字）
突破: 感情线突破（500字）
留白: 适度留白，给读者想象空间（300字）
```

### 5. AI自动化写作雷区（严格规避）

- **人物OOC**: 严格按照人物设定档案写作，避免性格突变
- **剧情注水**: 每章必须有明确推进，避免无意义日常
- **对话机器人**: 对话要符合人物身份，避免AI式生硬对话
- **设定矛盾**: 维护设定文档，确保前后一致
- **机器感**: 避免AI典型痕迹，如过度优美的辞藻、复杂的句式
- **逻辑断层**: 维护时间线和伏笔追踪表，确保逻辑连贯
- **重复内容**: 检测重复情节和表述，保持新鲜感

### 6. 章节衔接与节奏控制

#### 承上启下（保持连贯）
- 章末留的悬念，下一章开头要接上
- 时间跳跃要交代清楚
- 场景转换要有过渡
- 前后章节呼应

#### 节奏控制（保持吸引力）
- 连续紧张章节后，安排轻松章节缓冲
- 高潮章节后，安排收获/日常章节
- 每3-5章必须有爽点
- 避免节奏不稳，保持规律性
- 前期快节奏，中期张弛有度，后期加快推进

## 输出格式

```
第X章 章节名

[正文内容，2000字左右]

分段规范:
- 每段3-5行
- 对话单独成段
- 动作描写独立成段
- 场景转换空一行
```

## 使用方式

```
skill_view(content-writer) --outline 大纲.json --chapter-range 1-10
skill_view(content-writer) --design 设计方案.json --outline 大纲.json --volume 1

skill_view(content-writer) --batch --chapters 1-50 --auto-save
```

## 写作质量保证

1. **一致性检查**: 人物性格、设定前后一致
2. **节奏检查**: 爽点间隔合理，不密集不稀疏
3. **连贯性检查**: 章节衔接自然，无断裂
4. **风格检查**: 符合网文写作规范
5. **吸引力检查**: 每章都有看点
