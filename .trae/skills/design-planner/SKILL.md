# 全维度设计清单 Skill

## 功能概述

本 skill 基于选题分析结果，生成小说的全维度设计清单，包括人物设定、世界观构建、力量体系、情节架构、金手指设计等核心要素，为后续大纲规划和内容撰写提供完整的设计蓝图。

## 触发条件

- 用户要求进行小说全维度设计
- 选题确定后需要详细设计方案
- 需要创建人物设定、世界观、力量体系等
- 工作流接收到选题分析结果后自动触发

## 设计维度

### 1. 人物设计体系

#### 主角设计
- **基本信息**: 姓名、年龄、性别、外貌特征
- **性格特质**: 核心性格、隐藏性格、性格缺陷
- **背景故事**: 出身、经历、创伤、动机
- **能力设定**: 初始能力、成长路径、特殊天赋
- **金手指**: 类型、规则、限制、成长机制
- **人物弧光**: 起始状态、转变契机、最终成长
- **读者共鸣点**: 代入感设计、共情点、爽点

#### 配角设计
- **核心配角** (3-5个): 功能定位、与主角关系、独立故事线
- **功能性配角**: 导师、伙伴、对手、反派、爱情线
- **反派设计**: 动机、能力、与主角对立点、反派魅力
- **群像设计**: 势力分布、人物关系网

#### 人物关系矩阵
```
主角 <-> 导师 (指导/背叛)
主角 <-> 伙伴 (成长/竞争)
主角 <-> 反派 (对立/镜像)
主角 <-> 爱情线 (吸引/阻碍)
```

### 2. 世界观构建

#### 世界框架
- **世界类型**: 现代都市、异世界、未来科幻、历史架空
- **世界规模**: 单城市、多国、多大陆、宇宙级
- **世界规则**: 物理法则、社会结构、文化体系
- **时代背景**: 古代、现代、未来、特定历史时期

#### 社会体系
- **权力结构**: 政府、组织、家族、宗门
- **经济体系**: 货币、贸易、资源分布
- **阶级划分**: 平民、贵族、修士、异能者
- **文化特色**: 信仰、习俗、禁忌、传说

#### 地图设计
- **核心区域**: 主要故事发生地
- **探索区域**: 后续扩展空间
- **禁区/秘境**: 特殊功能区域
- **区域关联性**: 交通、势力、资源

### 3. 力量体系设计

#### 力量等级
```
等级1 (入门) -> 等级2 (进阶) -> 等级3 (高手) -> 等级4 (宗师) -> 等级5 (巅峰)
```

#### 力量类型
- **修炼体系**: 功法、境界、瓶颈
- **技能系统**: 主动技能、被动技能、天赋技能
- **装备系统**: 武器、防具、辅助道具
- **特殊能力**: 血脉、体质、天赋

#### 力量平衡
- **上限设定**: 最高战力天花板
- **克制关系**: 相生相克
- **成长曲线**: 前期、中期、后期节奏
- **越级挑战**: 条件、代价、合理性

### 4. 金手指设计

#### 金手指类型
- **系统流**: 任务系统、抽奖系统、签到系统
- **重生流**: 预知未来、弥补遗憾、抢占先机
- **穿越流**: 知识碾压、降维打击、文化碰撞
- **奇遇流**: 神器传承、高人指点、血脉觉醒
- **选择流**: 人生选择、因果改变、命运分支

#### 金手指规则
- **获取方式**: 如何获得、触发条件
- **使用限制**: 冷却时间、消耗代价、使用条件
- **成长机制**: 升级路径、解锁条件、突破方式
- **平衡设计**: 不能无敌、需要代价、有限制

### 5. 情节架构设计

#### 主线设计
- **核心冲突**: 主角vs反派、主角vs命运、主角vs自我
- **终极目标**: 最终要达成的目的
- **驱动引擎**: 推动故事前进的核心动力

#### 副线设计
- **爱情线**: 相遇、发展、阻碍、结局
- **友情线**: 结盟、信任、考验、坚守
- **成长线**: 能力成长、心理成长、认知成长
- **复仇线/使命线**: 起因、过程、结果

#### 爽点设计
- **打脸爽**: 被轻视后逆袭打脸
- **升级爽**: 突破瓶颈、实力飞跃
- **收获爽**: 获得宝物、技能、人脉
- **碾压爽**: 以强欺弱、碾压对手
- **反转爽**: 意料之外、情理之中

### 6. 标签与卖点设计

#### 核心标签
- 题材标签: 玄幻、都市、仙侠等
- 风格标签: 轻松、热血、暗黑、搞笑
- 元素标签: 系统、重生、穿越、群像

#### 卖点提炼
- **第一卖点**: 最吸引人的核心亮点
- **第二卖点**: 辅助吸引力
- **差异化卖点**: 与竞品小说的区别

## 使用方式

```
/design-planner --track 都市异能 --protagonist-name 林辰
/design-planner --input 选题分析.json --output 设计方案.json
/design-planner --focus 人物设计 --track 玄幻
/design-planner --focus 世界观 --world-type 异世界
```

## 设计原则

1. **一致性**: 各维度设计相互呼应，不矛盾
2. **可扩展性**: 预留后续发展空间
3. **读者导向**: 以读者爽点和代入感为核心
4. **差异化**: 避免同质化，突出特色
5. **可行性**: 设计要能在实际写作中落地
