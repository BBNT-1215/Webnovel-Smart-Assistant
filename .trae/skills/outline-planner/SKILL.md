# 大纲规划 Skill

## 功能概述

本 skill 基于全维度设计方案，生成结构化的小说大纲，包括卷纲设计、章纲规划、情节节奏安排、高潮布局等，为全文撰写提供详细的写作指导。

## 触发条件

- 用户要求生成小说大纲
- 设计方案完成后需要细化为章节规划
- 需要安排情节节奏和高潮点
- 工作流接收到设计方案后自动触发

## 大纲层级结构

### 1. 总纲设计

#### 故事概览
- **一句话简介**: 用一句话概括故事核心
- **核心卖点**: 2-3个吸引读者的亮点
- **故事基调**: 轻松/热血/暗黑/搞笑
- **预期字数**: 总字数规划（通常100-500万字）

#### 故事主线
```
起因 (1-10%) -> 发展 (10-50%) -> 高潮 (50-80%) -> 结局 (80-100%)
```

#### 核心冲突
- **外部冲突**: 主角vs反派、主角vs环境
- **内部冲突**: 主角vs自我、信念冲突
- **关系冲突**: 主角vs亲人/朋友/爱人

### 2. 卷纲设计

#### 分卷原则
- **每卷字数**: 20-50万字
- **卷内完整性**: 每卷有独立的小高潮和结局
- **卷间关联**: 上下卷之间有承上启下
- **节奏递进**: 越往后节奏越快、冲突越强

#### 单卷结构 (以30万字为例)
```
第1-5万字: 铺垫期 - 引入新设定、新人物
第5-15万字: 发展期 - 冲突升级、能力成长
第15-25万字: 高潮期 - 大战爆发、关键转折
第25-30万字: 收尾期 - 解决冲突、铺垫下一卷
```

#### 卷纲模板
```
第X卷: [卷名]
- 核心事件: [本卷主要发生什么]
- 主要目标: [主角本卷要达成什么]
- 主要对手: [本卷主要反派/阻碍]
- 能力成长: [本卷主角实力提升到什么程度]
- 感情进展: [本卷感情线推进到什么程度]
- 高潮场景: [本卷最精彩的情节]
- 悬念设置: [本卷结尾留下什么悬念]
```

### 3. 章纲设计

#### 章节节奏 (以2000字/章为例)
- **开头 (0-300字)**: 钩子/悬念/冲突引入
- **发展 (300-1500字)**: 情节推进/冲突升级
- **高潮 (1500-1800字)**: 爽点爆发/反转
- **结尾 (1800-2000字)**: 留悬念/勾读者

#### 章节类型
- **铺垫章**: 世界观介绍、人物出场、设定说明
- **冲突章**: 矛盾激化、战斗、对抗
- **爽点章**: 打脸、碾压、逆袭、收获
- **过渡章**: 场景转换、时间跳跃、信息交代
- **高潮章**: 大决战、关键转折、情感爆发
- **悬念章**: 章末留悬念、勾读者追更

#### 单章结构模板
```
第X章: [章名]
- 场景: [在哪里发生]
- 人物: [哪些人物出场]
- 事件: [发生什么事]
- 冲突: [矛盾点是什么]
- 爽点: [本章节的爽点/看点]
- 结尾悬念: [章末留什么勾子]
```

### 4. 情节节奏设计

#### 爆款节奏模型
```
开篇节奏（1-30万字）:
- 极快节奏，爽点密集，冲突频繁
- 3章内出金手指，每章至少1个小爽点
- 10章内有打脸情节，30万字内换地图

中期节奏（30-100万字）:
- 稳定节奏，快慢结合，深入展开
- 2-3章一个爽点，深化人物和世界观
- 设置中期高潮，处理支线情节

后期节奏（100万字+）:
- 收放自如，加快主线，收束支线
- 加快主线推进，处理伏笔和支线
- 准备最终高潮，给读者满意结局

张弛有度: 紧张章节(5章) -> 轻松章节(2章) -> 紧张章节(5章) -> 大高潮(10章) -> 轻松章节(3章)
爽点密度: 小爽点每3000字一次，中爽点每1万字一次，大爽点每3-5万字一次
小高潮: 每10-15章安排一个小高潮
大高潮: 每50-80章安排一个大高潮
```

#### 爽点分布策略
- **开篇3章定生死**: 第1章出冲突，第2章给金手指，第3章打脸爽
- **前期快节奏**: 前30万字，每章必有看点，3-5章一个大爽点
- **中期稳定**: 30-100万字，2-3章一个爽点，张弛有度
- **后期冲刺**: 100万字后，加快节奏，收束情节

#### 情绪曲线管理
```
开篇期(1-3章): 冲突爆发 -> 绝境 -> 金手指觉醒 -> 逆袭 -> 暂时胜利 -> 新危机
发展期(4-100章): 小爽点循环 -> 能力提升 -> 更强对手 -> 暂时挫折 -> 重大突破 -> 更大目标
高潮期(100章+): 主线加速 -> 多线汇聚 -> 最终对决 -> 圆满结局
```

#### 爽点排布表（爆款标准）
| 章节区间 | 爽点类型 | 强度 | 作用 | 备注 |
|----------|----------|------|------|------|
| 第1章 | 冲突爆发 | ★★★★★ | 抓住读者 | 退婚、背叛、羞辱等 |
| 第2章 | 金手指觉醒 | ★★★★★ | 给予希望 | 系统、重生、传承等 |
| 第3章 | 初次打脸 | ★★★★★ | 爽感爆发 | 让看不起主角的人傻眼 |
| 第4-10章 | 小试身手 | ★★★ | 验证能力 | 逐步展现实力 |
| 第11-20章 | 小打脸 | ★★★★ | 积累爽感 | 连续打脸小反派 |
| 第21-30章 | 实力突破 | ★★★★ | 成长爽 | 突破境界/获得新能力 |
| 第31-50章 | 中等打脸 | ★★★★★ | 中等高潮 | 打脸中等反派 |
| 第51-100章 | 持续升级 | ★★★★ | 爽感循环 | 保持追读 |
| 第100章+ | 大高潮 | ★★★★★ | 长期爽感 | 处理大反派 |

### 5. 伏笔与悬念设计

#### 伏笔类型
- **明伏笔**: 读者能察觉的暗示
- **暗伏笔**: 隐藏的线索，后续揭晓
- **误导伏笔**: 故意引导读者错误推测
- **回收伏笔**: 前面埋的线索后面使用

#### 伏笔管理表
```
伏笔ID: F001
- 埋设章节: 第5章
- 伏笔内容: 主角手上的神秘胎记
- 回收章节: 第50章
- 回收方式: 胎记觉醒，揭示身世
- 伏笔类型: 身世之谜
```

#### 悬念设计
- **章末悬念**: 每章结尾留勾子
- **卷末悬念**: 每卷结尾留大悬念
- **跨卷悬念**: 贯穿多卷的长期悬念

## 使用方式

```
/outline-planner --design 设计方案.json --volumes 5 --chapters-per-volume 100
/outline-planner --track 都市异能 --focus 第一卷大纲
/outline-planner --input 设计清单.json --output 大纲.json
/outline-planner --adjust-pacing --fast-start --volume 1
```

## 大纲设计原则

1. **强开头**: 前3章必须有吸引力
2. **快节奏**: 前期节奏要快，不拖沓
3. **密爽点**: 爽点要密集但不能泛滥
4. **有悬念**: 时刻勾住读者好奇心
5. **可扩展**: 大纲要留有调整空间
6. **可执行**: 每个章节都要有明确的写作指导
